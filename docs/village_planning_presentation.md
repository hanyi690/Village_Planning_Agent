# 项目概述

## 核心定位

将村庄规划从"人工经验驱动"升级为"AI智能辅助"

## 主要功能

- **三层递进规划**：现状分析 → 规划思路 → 详细规划
- **28维度覆盖**：全面涵盖村庄规划各专业领域（12+4+12）
- **RAG知识增强**：法规条文与技术指标智能注入
- **实时交互**：Token级流式输出，支持人工审查与修订反馈

# 三层规划: LangGraph 编排

## 子图架构

```
MainGraph (主流程编排)
├── Layer1Subgraph (现状分析子图)
│   ├── init_node → 初始化状态
│   ├── preload_node → 知识预加载
│   ├── analysis_nodes → 12维度并行 (Send机制)
│   └── merge_node → 汇总输出
├── Layer2Subgraph (规划思路子图)
│   └── 波次调度执行 → 4维度顺序执行
└── Layer3Subgraph (详细规划子图)
    └── 波次调度执行 → 12维度两波次
```

## 数据流转

- **层间传递**：上层层级输出 → 下层输入上下文
- **Checkpoint**：每层完成后自动保存状态，支持回退

# Layer 3: 项目库优化

## 问题

项目库需引用11份报告（~50,000字符），Token消耗巨大

## 解决方案：影子缓存

```
原始报告（~50,000字符）
        ↓ 提取项目摘要
影子缓存（~5,000字符）
        ↓
节省 90% Token
```

## 提取内容

项目名称、建设内容、投资规模、建设位置、实施阶段

# 波次调度机制

## 核心原理

1. **依赖配置**：定义维度间依赖关系
2. **波次计算**：拓扑排序自动计算执行波次
3. **动态路由**：运行时根据已完成维度决定下一步

## 波次计算算法

```
def calculate_wave(dimension, dependencies):
    # 无同层依赖 → Wave 1
    if not dependencies:
        return 1
    # 依赖多个维度 → 取最大 Wave + 1
    return max(dep.wave for dep in dependencies) + 1
```

## 运行时路由

```
检查已完成维度 → 计算可执行维度(Wave匹配) → 分发到对应节点
```

# RAG: 文档处理流程

## 切片策略

```
RecursiveCharacterTextSplitter
├── chunk_size: 2500 字符（教科书可调至 4000）
├── chunk_overlap: 500 字符
└── 分隔符优先级: 段落 → 句子 → 词
```

## 向量化引擎

- **默认方案**：ChromaDB + BAAI/bge-small-zh-v1.5（本地）
- **云端方案**：阿里云 text-embedding-v4（API 调用）
- **向量维度**：512（本地）/ 1024（云端）

## 检索流程

```
用户查询 → 向量编码 → 余弦相似度计算 → Top-K 匹配 → 上下文扩展 → 格式化输出
```

# RAG: 知识搜索详解

## 三种上下文模式

| 模式 | 返回内容 | 适用场景 |
|------|----------|----------|
| minimal | 仅匹配片段 | 快速查找，节省Token |
| standard | 片段+300字上下文 | 常规检索（默认） |
| expanded | 片段+500字上下文 | 需要更多上下文时 |

## 检索流程

```
用户查询 → 向量匹配 → 返回Top-K片段 → 扩展上下文 → 格式化输出
```

# GIS: 空间数据集成

## 功能定位

将村庄空间数据注入规划分析，提供真实地理依据

## 数据来源

| 数据类型 | 来源 | 规划用途 |
|----------|------|----------|
| 地形数据 | DEM数字高程 | 坡度分析、选址评估 |
| 用地现状 | 国土调查数据 | 用地平衡、结构调整 |
| 设施分布 | POI数据 | 服务半径、布局优化 |
| 路网数据 | OSM/交通部门 | 交通分析、可达性 |

## 实现逻辑

```
GIS数据库 → 格式转换(GeoJSON) → 结构化提取 → 注入规划上下文
```

# GIS: 规划图件生成

## 功能定位

基于规划文本自动生成空间布局图件

## 可生成图件

| 图件类型 | 内容 | 用途 |
|----------|------|------|
| 用地规划图 | 各类用地范围、边界 | 审批报批 |
| 设施布局图 | 公服、基础设施位置 | 建设指导 |
| 道路系统图 | 路网等级、断面 | 工程实施 |
| 景观格局图 | 绿地、景观节点 | 风貌引导 |

## 实现逻辑

```
规划文本 → LLM结构化提取 → GeoJSON → GIS渲染引擎 → 图件输出
```

## 技术路径

- 文本解析：LLM 提取用地/设施/道路等空间要素
- 空间映射：要素属性 → GeoJSON 几何对象
- 图件渲染：GeoJSON + 底图 → QGIS/Mapbox → 输出
