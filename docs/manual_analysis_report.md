## 六、核心结论

### 6.1 RAG 效果分维度评估

| 维度                | RAG 效果评估           | 原因分析                           |
| ------------------- | ---------------------- | ---------------------------------- |
| ecological          | ✅ **效果最好**  | 知识库有生态修复工程，匹配度高     |
| land_use_planning   | ⚠️ **部分有效** | 上位规划衔接好，但数值需验证       |
| disaster_prevention | ⚠️ **部分有效** | 有GB号但缺年份，优于无引用         |
| heritage            | ⚠️ **效果持平** | 双方都正确引用国家法律             |
| infrastructure      | 🔴 **效果最差** | 知识库缺技术标准，导致大量虚构     |

### 6.2 幻觉产生机制分析

**RAG 知识注入悖论**:

1. 知识库注入的是《梅州市国土空间总体规划》（宏观规划）
2. 模型需要引用具体技术标准（给排水、抗震等）
3. 知识库中**缺少**具体技术标准内容
4. 模型**幻觉**出"应该存在"的标准名称来填补空白
5. 结果：宏观规划引用正确，但技术细节引用虚构

**RAG OFF 反而正确**:

1. 无知识注入，模型完全依赖内部知识
2. 内部训练数据包含GB5749等常见国家标准
3. 直接引用真实的GB号，不虚构"专业准则"
