# Claude Custom Instructions

## 项目概述

这是一个基于 LangGraph 和 LangChain 的村庄规划智能体系统，采用分层架构和交互式工作流实现专业的村庄规划辅助。

**当前版本：v4.2.0**

## 核心架构理解

### 技术栈
- **LangGraph**: 用于构建分层图架构，管理状态流转和节点调度
- **LangChain**: 提供 LLM 集成、提示词管理和工具调用能力
- **架构模式**:
  - 分层图架构（主图 + 三层子图）
  - Map-Reduce 并行处理模式
  - 依赖链驱动的动态调度

### 三层架构
1. **Layer 1 (现状分析)**: 12个维度并行分析
2. **Layer 2 (规划思路)**: 4个维度并行分析
3. **Layer 3 (详细规划)**: 10个维度分波次执行

### 1. 编程语言和风格
- 使用 **Python** 作为主要编程语言

### 2. 图结构设计原则
使用 LangGraph 构建图时，遵循以下规范：

```python
# 优先使用 StateGraph 定义图结构
from langgraph.graph import StateGraph, END, START
from typing import TypedDict, Annotated
from langgraph.graph.message import add_messages

# 定义状态（必须使用 TypedDict）
class GraphState(TypedDict):
    """图状态定义"""
    messages: Annotated[list, add_messages]  # 消息历史必须包含
    current_step: str                          # 当前步骤
    # ... 其他状态字段

# 创建图
builder = StateGraph(GraphState)
builder.add_node("node_name", node_function)
builder.add_edge(START, "node_name")
builder.add_conditional_edges("node_name", route_function)
graph = builder.compile()
```

### 3. 节点函数规范
每个节点函数必须：

```python
def node_function(state: GraphState) -> Dict[str, Any]:
    """
    节点函数的标准签名

    Args:
        state: 当前图状态

    Returns:
        包含状态更新的字典，只包含需要更新的字段
    """
    # 节点逻辑
    result = do_something()

    # 返回状态更新（只返回需要更新的字段）
    return {
        "current_step": "next_step",
        "output": result
    }
```

### 4. 路由函数规范
条件路由函数必须返回明确的字符串：

```python
def route_function(state: GraphState) -> Literal["option1", "option2", "end"]:
    """
    条件路由函数

    Args:
        state: 当前图状态

    Returns:
        路由目标节点名称或 END
    """
    if condition_met:
        return "option1"
    else:
        return "end"  # 使用 "end" 而非 END
```

### 11.1 适配器开发规范
适配器用于隔离外部专业计算库（如 geopandas、networkx）的复杂性。

**核心原则**：
- **明确失败优于静默模拟**：缺少依赖或数据时，必须返回清晰的错误
- **保持测试能力**：MockAdapter 保留，但明确标注为测试专用
- **清晰错误信息**：当缺少依赖或数据时，提供明确的错误提示

**适配器模板**：

```python
from .base_adapter import BaseAdapter, AdapterResult, AdapterStatus

class MyAdapter(BaseAdapter):
    """
    我的适配器

    支持的分析类型：
    1. analysis_type_1
    2. analysis_type_2
    """

    def __init__(self, config: Optional[Dict[str, Any]] = None):
        super().__init__(config)
        self._dependency_available = False

    def validate_dependencies(self) -> bool:
        """验证外部依赖是否可用"""
        try:
            import external_library
            self._dependency_available = True
            return True
        except ImportError:
            return False

    def execute(self, analysis_type: str = "analysis_1", **kwargs) -> AdapterResult:
        """执行分析"""
        # 检查依赖
        if not self._dependency_available:
            return AdapterResult(
                success=False,
                status=AdapterStatus.FAILED,
                data={},
                metadata={},
                error="分析需要 external_library 库。请安装: pip install external_library"
            )

        # 检查必需参数
        required_param = kwargs.get("required_param")
        if not required_param:
            return AdapterResult(
                success=False,
                status=AdapterStatus.FAILED,
                data={},
                metadata={},
                error="缺少必需参数: required_param。请提供..."
            )

        # 执行真实分析
        result = self._analyze_real(**kwargs)
        return AdapterResult(
            success=True,
            status=AdapterStatus.SUCCESS,
            data=result,
            metadata={"analysis_type": analysis_type}
        )

    def _analyze_real(self, **kwargs) -> Dict[str, Any]:
        """真实分析逻辑"""
        import external_library
        # 实现分析逻辑
        return {"result": "data"}
```

**禁止事项**：
- ❌ 不要添加 `fallback_to_mock` 配置
- ❌ 不要实现 `_analyze_*_mock` 方法
- ❌ 不要使用默认值生成模拟数据

### 17. 禁止事项
- ❌ 不要在状态中使用不可序列化的对象
- ❌ 不要在节点函数中修改全局变量
- ❌ 不要在路由函数中执行耗时操作
- ❌ 不要使用阻塞式 I/O 操作（使用异步或线程池）
- ❌ 不要在循环中直接调用 LLM（批处理或并行化）
- ❌ **不要在生产环境中使用 MockAdapter** - MockAdapter 仅用于测试
- ❌ **不要在适配器中使用模拟数据** - 所有适配器必须要求真实数据
- ❌ **不要为适配器添加 fallback_to_mock 配置** - 系统已移除模拟数据支持

- 使用 **波次调度** 优化维度执行顺序
- 启用 **Checkpoint** 持久化避免重复计算

## 关键设计原则

1. **单一职责**: 每个节点只负责一个明确的任务
2. **状态不可变性**: 节点函数不直接修改传入的状态对象
3. **依赖清晰**: 通过 dimension_mapping 明确依赖关系
4. **可测试性**: 核心逻辑与框架代码分离
5. **可恢复性**: 支持 Checkpoint 持久化和恢复

## 常用代码模式

### 模式 1: 节点基类
```python
class BaseNode:
    """节点基类，提供通用功能"""
    def __init__(self, name: str):
        self.name = name
        self.logger = get_logger(__name__)

    def __call__(self, state: Dict[str, Any]) -> Dict[str, Any]:
        """调用节点执行方法"""
        return self.execute(state)

    def execute(self, state: Dict[str, Any]) -> Dict[str, Any]:
        """子类实现"""
        raise NotImplementedError
```

### 模式 2: 工具基类
```python
class BaseTool:
    """工具基类"""
    def __init__(self):
        self.logger = get_logger(__name__)

    def run(self, **kwargs) -> Dict[str, Any]:
        """工具执行入口"""
        try:
            return self._execute(**kwargs)
        except Exception as e:
            self.logger.error(f"工具执行失败: {e}")
            return {"success": False, "error": str(e)}

    def _execute(self, **kwargs) -> Dict[str, Any]:
        """子类实现"""
        raise NotImplementedError
```

### 模式 3: 规划器基类
```python
class BasePlanner:
    """规划器基类"""
    def __init__(self, dimension: str):
        self.dimension = dimension
        self.logger = get_logger(__name__)
        self.llm = create_llm()

    def execute(self, state: Dict[str, Any]) -> Dict[str, Any]:
        """执行规划"""
        # 筛选状态
        filtered_state = self._filter_state(state)

        # 构建 Prompt
        prompt = self._build_prompt(filtered_state)

        # 调用 LLM
        response = self.llm.invoke(prompt)

        return {
            self.dimension: response.content
        }

    def _filter_state(self, state: Dict[str, Any]) -> Dict[str, Any]:
        """筛选状态（子类可覆盖）"""
        return state
```

## 注意事项

1. **中文环境**: 系统主要面向中文用户，所有输出和提示使用中文
