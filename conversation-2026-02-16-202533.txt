  问题分析报告

  根本问题

  问题核心: _token_callback_factory
  通过两种路径传递，但子图节点中的获取逻辑存在错误。

  问题 1: 子图节点中获取 token 回调工厂的方式错误

  位置: src/nodes/subgraph_nodes.py:94-99

   # ❌ 错误代码
   on_token_callback = None
   if hasattr(state, "config") and state.config.get("configurable"):
       token_callback_factory = state.config["configurable"].get(
   "_token_callback_factory")

  问题:
   - state 是一个 TypedDict 字典，不是对象
   - hasattr(state, "config") 对于字典永远返回 False
   - 因此 on_token_callback 永远是 None，流式回调从未触发！

  正确做法:
   - 工厂已经通过 Send 机制传递到 dimension_state 中
   - 应该从 state.get("_token_callback_factory") 获取（已经传递过来了）

  问题 2: 前端消息类型使用错误

  位置: frontend/src/components/chat/ChatPanel.tsx:213-217

   // 创建了错误的消息类型
   addMessage({
     type: 'layer_completed' as const,  // 应该是 'dimension_report' 
   用于流式显示
     ...
   });

  问题:
   - dimension_report 消息类型专门用于流式显示单个维度内容
   - layer_completed 消息类型用于显示整个层级完成的报告
   - 当前代码在收到第一个 token 时创建 layer_completed 消息，但更新
     dimensionReports 字段

  问题 3: 前端依赖 SSE 事件，但后端未正确发送

  由于问题 1，后端的 dimension_delta 事件从未发送，前端：
   1. 无法实时显示维度内容
   2. 只能等待 layer_completed 事件（层级完全完成后）
   3. 用户看到的是"不断加载"但没有内容更新

  方案 A: 修复 token 回调获取方式（推荐）

  修改 src/nodes/subgraph_nodes.py:

   # ✅ 正确代码
   on_token_callback = None
   token_callback_factory = state.get("_token_callback_factory")
   if token_callback_factory:

       on_token_callback = token_callback_factory(layer, dimension_key)

  优点: 最小改动，直接利用已有的传递机制
